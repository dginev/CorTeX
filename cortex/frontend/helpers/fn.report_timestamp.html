pub fn report_timestamp() -> String
Expand description

The “generated at” timestamp shown in report footers: the server’s local date and time to the minute, suffixed with the time-zone abbreviation — e.g. Sat, 13 Jun 2026 22:57 EDT.

chrono’s %Z only renders the numeric UTC offset (-04:00) for Local, so the abbreviation comes from the C library’s strftime %Z, which reads the OS time-zone database and is DST-correct. Falls back to chrono’s offset rendering if the platform yields no abbreviation, so the zone is never blank.
